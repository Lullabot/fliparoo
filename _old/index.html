Display & Queue list:

The best practice for Fliparoo is to use two sets of elements. One is a "display" set (such as a <li> list) which acts as a container for the elements being shown. The other is a "queue" set which acts as a stockpile. By hiding the queue list with CSS, there will not be a FOUC (flash of unstyled content) between the time that the page begins to render and the time that the Fliparoo script runs. Perhaps more important is that if Javascript doesn't run on the page, the queue list elements will remain hidden and the fallback will be the original non-animating display elements.

The jQuery selectors for the display and queue lists should reference the individual items within the set – the <li> items rather than the <ul> for example. In more advanced examples, this will allow us to non-contiguous display or queue items.

Single Combined Display/Queue list:

Fliparoo can also function with a single set of elements. This is obviously a simpler way to go, however In this case, you'll want to define a 'displayCount' property so that Fliperoo knows how many of the items to use for display and how many items to hide and put into the queue. This example also shows several other Fliperoo options including X-axis rotation, randomization of the queue, and custom easing.

Kittens:

Usually, Fliparoo chooses a random display order which it repeats to display items in the queue. It is also possible to choose 'displayOrder: forward' or 'displayOrder: reverse' to iterate through the display set. By combining this with the 'setDelay' parameter, you can get a train-station-display type transition with a flutter of activity as all display items change and then pausing for visitors to parse the new information. This example uses the 'flipFlop' parameter to create a Jacob's Ladder effect without any delay between flips.
